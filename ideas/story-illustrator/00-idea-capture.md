# COMFYUI-STORY-ILLUSTRATOR — Idea Capture

## One-line summary

Batch-конвейер генерации серий иллюстраций к рассказам через ComfyUI API
с сохранением характера персонажей (consistency) между сценами.

## Problem / purpose

Работа «с наскока» привела к каше: перемешались стили (реализм/иллюстрация),
базовые модели, эксперименты и результаты. Нужен чёткий, воспроизводимый
конвейер, которым можно работать из любого агента (Hermes, Cursor chat,
Cursor terminal) по единому ТЗ.

## Initial bullet points (согласованные принципы)

- **Один движок, стили — конфигурации.** Единый раннер (ComfyUI API,
  stdlib-only Python). Стиль = профиль: модель + LoRA + принципы промпта +
  негативы + параметры. Не плодить отдельные скрипты на стиль.
- **Профили стилей:** реализм / иллюстрация / абстракция / NSFW.
- **Выходы по осям «рассказ → стиль/модель → сцена»:**
  `sessions/<рассказ>/<стиль>/<сцена>/v<NN>_seed<...>.png`.
  Сравнение моделей = две соседние папки одного рассказа.
- **Эксперимент ≠ продукт:** пробы, калибровки — в `experiments/`
  с коротким выводом; в основные папки — только отобранное.
- **Знания по стилям** — разделы одного KNOWLEDGE_BASE.md.
- **Consistency персонажей:** character sheets + IP-Adapter
  (канонический референс на персонажа, калиброванный вес адаптера).

## Key decisions (interview 2026-07-12)

1. **Продукт** — и то и другое: конвейер — цель, «В Древней Греции полдень
   горяч» — первый полигон.
2. **MVP-стили** — два сразу: реализм + иллюстрация (сравнение бок о бок
   с первого дня). Абстракция и NSFW — потом, как новые конфиги.
3. **Вход конвейера** — гибрид: агент предлагает сцены/промпты, Yuri правит
   и утверждает файл сцен, затем генерация.
4. **Consistency в MVP** — базовая: IP-Adapter держит одного главного героя
   сцены, остальные через детальные промпты. Полная (Regional IP-Adapter,
   несколько узнаваемых лиц в кадре) — отдельной фазой в роадмапе.
5. **Критерий приёмки MVP** — оба: (а) Греция готова — 5 сцен × 2 стиля,
   каноничные герои, отобранные финалы; (б) тест на новизну — новый рассказ
   проходит конвейер без правки кода, только новым файлом сцен.
