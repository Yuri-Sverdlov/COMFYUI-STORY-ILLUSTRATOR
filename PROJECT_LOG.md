## 2026-07-09 — ПИВОТ: с ComfyUI на облачную генерацию (OpenRouter)

**Архитектор:** чат.

- **Причина:** M5 (полный прогон Греции, 40 картинок) дал брак — все
  погрудные портреты вместо сцен. IP-Adapter (weight 0.85, preset PLUS)
  переносит с референса не только лицо, но и кадрирование. Негативы
  «NOT a portrait» CLIP не понимает. Конфликт «консистенция vs композиция» —
  фундаментальная слабость локального SDXL-пайплайна.
- **Решение Yuri:** перейти на OpenRouter Image API
  (google/gemini-3-pro-image, ~$0.03–0.06/картинка). Модель принимает
  референсы прямо в запросе и сама держит лица/одежду — без IP-Adapter,
  ControlNet, негативов.
- **TASK-002 закрыт** со статусом «остановлен: пивот на облако».
  M5 признан браком, M6–M7 отменены.
- **ComfyUI-пайплайн НЕ удалён** — остаётся для NSFW и локальных экспериментов.
- **Создан TASK-003:** генерация через OpenRouter Image API.
  Три шага: этюд (1 запрос, проверка гипотезы) → раннер → полный прогон.
  Выдаётся кодеру по одному шагу.

**Следующий шаг:** TASK-003 Шаг 1 — этюд облачной генерации
(см. `tasks/TASK.md`).
